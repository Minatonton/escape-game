## 1. ゲーム概要

| 項目 | 内容 |
|------|------|
| ジャンル | 2D ポイント&クリック 謎解き脱出ゲーム |
| プレイ人数 | 1人 |
| プレイ時間 | 約 3〜5 時間（1周目） |
| ターゲット層 | 謎解き・パズル好き、10代〜40代 |
| Steam カテゴリ | Puzzle, Adventure, Point & Click |

### コンセプト

> **「謎が、次の謎を呼ぶ。」**

プレイヤーは見知らぬ場所に閉じ込められ、部屋に散りばめられた手がかりを集め、論理的思考とひらめきで脱出を目指す。各部屋は独立したパズルを持ちながらも、ストーリーが繋がることで「なぜここに閉じ込められているのか」という大きな謎が明らかになっていく。

### ユニークセリングポイント (USP)

- **連鎖するパズル設計** — 一つの答えが別のパズルのヒントになる
- **環境ストーリーテリング** — 会話なしで世界観を伝えるビジュアル演出
- **ヒントシステム** — 詰まっても段階的なヒントで進められる
- **複数エンディング** — 選択と発見の順序で結末が変わる

## 2. ストーリー・世界観

> ※ 以下はサンプルプロット。正式なストーリーはここを書き換えて確定させる。

### 設定

舞台は廃墟と化した研究施設。かつて記憶に関する研究を行っていた「Mnemosyne研究所」の内部。プレイヤーキャラクターは目覚めると記憶がなく、施設から脱出しながら自分が何者かを探る。

### 主要キャラクター

| 名前 | 役割 | 概要 |
|------|------|------|
| *(主人公)* | プレイヤー | 記憶を失った人物。年齢・性別不明 |
| *(博士)* | 謎の声 | 施設内の音声ログを通じてガイドする |
| *(A.I.)* | 対立者 | 施設を管理する人工知能。プレイヤーの脱出を妨害 |

### ストーリー構造

```
プロローグ（目覚め）
    │
    ├── チャプター 1: 研究室エリア（3部屋）
    │       └── ミッドポイント: 最初の記憶フラッシュバック
    │
    ├── チャプター 2: 実験棟エリア（4部屋）
    │       └── ミッドポイント: 博士の正体に関する手がかり
    │
    ├── チャプター 3: 制御室エリア（3部屋）
    │       └── クライマックス: A.I.との対峙
    │
    └── エンディング分岐（3種）
           ├── True End: 全記録を回収した場合
           ├── Normal End: 脱出のみ成功
           └── Bad End: A.I.に従った場合
```

## 3. コアメカニクス

### 3.1 ポイント&クリック操作

| 操作 | 動作 |
|------|------|
| 左クリック | オブジェクト調査 / 移動 / パズル操作 |
| 右クリック | アイテム詳細確認（インベントリ内） |
| オブジェクトホバー | カーソル変化 + オブジェクト名表示 |
| アイテム → オブジェクト | ドラッグ&ドロップ または アイテム選択→クリック |

### 3.2 調査システム

- **通常調査:** クリックでテキスト説明表示
- **詳細調査:** 一部オブジェクトはズームビューに入り、360°確認やパズル操作が可能
- **メモ機能:** 発見した番号・記号を自動でメモ帳に記録

### 3.3 アイテム合成

```
アイテムA + アイテムB → 新アイテムC
例: 錆びた鍵 + 潤滑油 → 使える鍵
```

### 3.4 ダイアログ / ログシステム

- 音声ログ（テキスト + 音声）をオブジェクトから発見
- ログは「ジャーナル」メニューからいつでも再生可能

## 5. 部屋・ステージ設計

### 部屋の基本要件

各部屋は以下の要素を持つ：

- **メインパズル:** 部屋の出口を開けるための中心的な謎（1〜2個）
- **サブ謎:** インベントリアイテムを取得するための補助的な謎（1〜3個）
- **ストーリーオブジェクト:** 世界観を補足する日記・写真・メモ（任意）
- **デッドオブジェクト:** 調査できるがパズルに関係しないフレーバー要素（数個）

### 部屋リスト（チャプター 1 例）

| 部屋ID | 部屋名 | 主なパズル | 獲得アイテム | 解除条件 |
|--------|--------|-----------|------------|---------|
| R001 | 目覚めの部屋 | 暗号ロック | IDカード | 4桁コード |
| R002 | 廊下 | 電気パズル | ワイヤー | 電力を繋ぐ |
| R003 | 研究室A | 色の配列 | 鍵 | カラーコード |

### 部屋移動ルール

- 各部屋は**前方向のみ進む**（バックトラックは原則なし）
- 一部の部屋は**別ルートに分岐**（エンディング分岐に影響）
- チャプタークリア時に**自動セーブ**

## 6. パズル設計指針

### パズルの種類

| タイプ | 説明 | 例 |
|--------|------|-----|
| 数字コード | 複数の手がかりから数字を導く | 本の特定ページの数字 |
| パターン認識 | 配列・色・形のパターンを解読 | カラーシーケンス |
| アイテム活用 | 正しいアイテムを正しい場所で使う | 鍵でロックを開ける |
| 環境操作 | 部屋の仕掛けを動かして解く | ミラーで光を反射 |
| 論理推理 | 条件から答えを絞り込む | 誰が何を持つか |
| テキスト解読 | 暗号文・記号を解く | シーザー暗号 |

### 設計の黄金律

1. **フェアであること** — 解答に必要な情報は必ず同じシーン内にある
2. **AHAモーメント** — 解けた瞬間の達成感を大切に設計する
3. **論理的整合性** — 「なぜそうなるのか」が説明できるパズルにする
4. **バランス** — 簡単・中程度・難しいパズルを混在させる（3:5:2の比率）

### 禁止事項

- 総当たりのみで解けるパズル
- ヒントなしの理不尽な謎（UI上で「謎すぎる」と感じる場合はリデザイン）
- パズルが実際のプレイを5分以上ブロックするような設計

## 7. インベントリシステム

### 仕様

- **スロット数:** 最大 16 スロット（チャプター単位でリセット可）
- **表示:** 画面下部のバー、またはボタンで展開するドロワー
- **操作:**
  - アイテムをクリック → 選択状態（カーソルに付く）
  - 選択状態でオブジェクトをクリック → 使用試行
  - インベントリ内のアイテム同士をドラッグ → 合成試行

### アイテムカテゴリ

| カテゴリ | 色ラベル | 例 |
|---------|---------|-----|
| 鍵・開錠系 | 黄色 | 鍵、IDカード、コード |
| 道具 | 青 | ドライバー、ワイヤー |
| 情報 | 緑 | メモ、写真、ログ |
| 素材 | グレー | 錆びたパーツ、布 |

## 8. UI / UX 設計

### 画面レイアウト

```
┌─────────────────────────────────────────┐
│  [メニュー]          [ヒント] [ジャーナル] │  ← HUD（常時表示）
├─────────────────────────────────────────┤
│                                         │
│              ゲーム画面                  │
│          （1920×1080 ベース）            │
│                                         │
├─────────────────────────────────────────┤
│  [🔑] [🔧] [📝] [  ] [  ] [  ] ...    │  ← インベントリバー
└─────────────────────────────────────────┘
```

### UX 原則

- **カーソルフィードバック:** インタラクト可能なオブジェクトでカーソル変化
- **誤操作防止:** アイテム使用失敗時は専用メッセージを表示（無言で無視しない）
- **オートセーブ:** チャプタークリアごとに自動保存、手動セーブも可能
- **字幕:** 全音声にテキスト字幕（アクセシビリティ対応）

### オプション設定

- 音量（BGM / SE / VO 個別）
- テキスト言語
- テキストサイズ（小/中/大）
- ヒント頻度（オフ/低/中/高）

## 9. アート方向性

### ビジュアルスタイル

- **解像度:** 1920×1080 ネイティブ（16:9）
- **スタイル:** 手描き風 / コミックシェーディング
- **カラーパレット:** チャプターごとにトーンを変える
  - Ch.1（研究室）: 寒色系、蛍光灯の白み
  - Ch.2（実験棟）: 暖色系、赤みがかった非常灯
  - Ch.3（制御室）: 緑色のモニター光

### アニメーション

- オブジェクトインタラクション時の小アニメーション（揺れ・点滅）
- チャプタークリア時のドアが開く演出
- キャラクターは静止画（Sprite）、感情表現はテキストで補足

## 10. サウンド設計

| 種類 | 方針 |
|------|------|
| BGM | 場所ごとに異なるアンビエント。テンションに応じて変化 |
| 環境音 | 冷却ファン、水滴、機械音など空間を演出する環境音 |
| SE | パズル操作音、アイテム取得音、ロック解除音 |

## 11. 難易度設計

### ヒントシステム

段階的に詳しくなる 3段階ヒント：

1. **ヒント1（ぼんやり）:** 「何かが隠されているかもしれない」
2. **ヒント2（方向性）:** 「この本棚をよく調べてみて」
3. **ヒント3（具体的）:** 「本の背表紙の数字を順番に並べると…」

ヒントはクールダウン制（60秒ごとに1回使用可能）。使用回数は記録されるが、エンディングには影響しない。

### 難易度オプション（任意実装）

- **イージー:** ヒント頻度高、インタラクト可能オブジェクトのハイライト表示
- **ノーマル:** 標準
- **ハード:** ヒントなし、メモ機能無効

## 12. ローカライズ

### 対応言語（優先度順）

| 優先度 | 言語 |
|--------|------|
| 必須 | 日本語 |
| 高 | 英語 |
| 中 | 中国語（簡体字） |
| 低 | 韓国語、ドイツ語 |

### 注意点

- フォントは各言語対応のものを用意
- テキストの文字数膨張（英語は日本語の1.3〜1.5倍になりやすい）を考慮してUIレイアウトを設計
- パズル内の文字・記号は言語依存しないデザインを優先
